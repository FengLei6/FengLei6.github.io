##一、序言
进程（process）：是并发执行的程序在执行过程中分配和管理资源的基本单位，是一个动态概念，竟争计算机系统资源的基本单位。每一个进程都有一个自己的地址空间，即进程空间或（虚空间）。进程至少有 5 种基本状态，它们是：初始态，执行态，等待状态，就绪状态，终止状态。

线程（Thread）：线程，是进程的一部分，一个没有线程的进程可以被看作是单线程的。线程有时又被称为轻权进程或轻量级进程，也是 CPU 调度的一个基本单位。

线程和进程的联系

一个进程可以包含多个线程，每个线程在一个进程中共同协作完成一个任务，一个进程中如果一个线程都不包含我们可以认为这个进程是单线程，因此这就很容易理解为什么说进程是c计算机系统分配资源的基本单位，因为计算机在执行某一个任务的时候就会把一整块的资源分派给进程。然后由进程里面的众多的线程完成这个资源的调度和使用。因此说线程是cpu调度和处理的基本单位。这就会引发一个问题，在同一个进程中的线程就会因为共同使用一块的资源，就会发生争抢的情况。这时就要有足够优质的控制机制来进行控制程序中线程的处理。

##二、多线程的创建

  1. 继承Thread类
  2. 实现runnable接口

继承Thread类

package com.Fl.Thread
public class MyThread extends Thread{
//重写run方法
public void run(){
//获得父类中的run方法的内容
super.run();
System.out.println("MyThread");
   }
}

实现runnable接口

注：如果想要创建的线程已经有自己的父类了，那么我们就不能再继承Thread类进行创建线程，因为Java只支持单继承。此时就要用实现runnable接口的方法来实现线程的创建。

package com.Fl.Thread
public class MyRunnable implement Runnable{
//重写run方法   
System.out.println("MyThread");
}

那么如何使用MyThread.java对象呢？

在Thread.java这个构造方法中只有Thread(Runnale r)和Thread(Runnale r，String name )可以传递Runnable接口，说明构造函数支持传入一个接口对象。

public class Run{
public static void main(String[] arge){
   Runnable runnable =new MyRunnable();
   Thread thread=new Thread(runnable);
   thread.start();
   }
}

构造函数Thread(Runnale r)不仅支持传入一个接口对象，还可以传入一个Thread对象。

##三、多线程的生命状态
注：在我们引入多线程给我们带来方便的同时，也引入了弊端和问题——线程的安全性。接下来就进行图解线程的安全性和线程状态等问题。

1.新建状态(new) : 线程对象被创建，此时线程就进入到新建状态。当我们new一个线程对象的时候，此时就是new状态。
2.就绪态(runnable):当线程对象被创建出来之后，我们就会调用它的start方法，此时，线程就会进入到就绪状态，此时的线程对象随时准备被CPU调用执行。
3.运行态(running):当线程对象在就绪状态的时候，它获得CPU的时间片，此时线程对象就进入到了运行状态，注意线程只能从就绪状态进入到运行状态。
4.阻塞状态(blocked):阻塞状态分为三种:
——等待阻塞:当处在运行状态的线程需要等待其他资源的时候就会调用wait()方法，进入阻塞状态。
——同步阻塞:线程在获取synchronized锁时失败，此时就会进入到同步阻塞状态。
——其他阻塞:当运行态的线程调用了sleep()或者jion()方法时，此时的线程也就进入阻塞状态。
5.死亡状态(DEAD):程序执行完，或者遇到异常，退出的执行。

###注:Java中的两种线程分类——用户线程和守护线程
守护线程其实就是一个给其他线程提供便利的线程。在JVM中只要有一个非守护线程没有结束，那么这个守护线程就不会结束。只有当JVM所有用户线程结束，守护线程就会结束。因此Daemon线程是为用户线程提供服务的线程，典型的守护线程就是GC(垃圾回收器)。

##四、Java多线程的常用方法和实例
—currentThread()返回对当前正在执行的线程对象的引用
—yield()暂停当前正在执行的线程对象，并执行其他线程。
—public static void sleep(long millis)throws InterruptedException:在指定的毫秒数内让当前正在执行的线程休眠（暂停执行），此操作受到系统计时器和调度程序精度和准确性的影响。该线程不丢失任何监视器的所属权。
—start():使该线程开始执行；Java 虚拟机调用该线程的 run 方法。
结果是两个线程并发地运行；当前线程（从调用返回给 start 方法）和另一个线程（执行其 run 方法）。 多次启动一个线程是非法的。特别是当线程已经结束执行后，不能再重新启动。
—public final boolean isAlive()测试线程是否处于活动状态。如果线程已经启动且尚未终止，则为活动状态
—setPriority(int newPriority)更改线程的优先级
—join
public final void join(long millis)
throws InterruptedException等待该线程终止的时间最长为 millis 毫秒
—isDaemon()测试该线程是否为守护线程。
—toString()返回该线程的字符串表示形式，包括线程名称、优先级和线程组。
—getId()返回该线程的标识符。线程 ID 是一个正的 long 数，在创建该线程时生成。线程 ID 是唯一的，并终生不变。线程终止时，该线程 ID 可以被重新使用。

###wait()方法和jion方法有什么不同之处？
1.sleep休眠时没有释放锁，而wait()释放了同步代码块，使得其他线程可以去执行被锁的同步代码块里面的类或者方法。
2.wait属于object的成员方法，一旦一个线程对象调用了wait方法，就必须要使用notify和notifyAll()方法唤醒该线程。

###实例
1.Thread 类中的start() 和 run() 方法有什么区别？
start()方法被用来启动新创建的线程，而且start()内部 调用了run()方法，这和直接调用run()方法的效果不一样。当你调用run()方法的时候，只会是在原来的线程中调用，没有新的线程启 动，start()方法才会启动新线程。
2.什么是线程安全？
代码所在的进程中有多个线程在同时运行，而这些线程可能会同时运行这段代码。如果每次运行结果和单线程运行的结果是一样的，而且其他的变量 的值也和预期的是一样的，就是线程安全的。
3.Java中如何停止一个线程？
Java提供了很丰富的API但没有为停止线程提供API。JDK 1.0本来有一些像stop(), suspend() 和 resume()的控制方法但是由于潜在的死锁威胁因此在后续的JDK版本中他们被弃用了，之后Java API的设计者就没有提供一个兼容且线程安全的方法来停止一个线程。当run() 或者 call() 方法执行完的时候线程会自动结束,如果要手动结束一个线程，你可以用volatile 布尔变量来退出run()方法的循环或者是取消任务来中断线程。
4.Java中notify 和 notifyAll有什么区别？
5.为什么wait, notify 和 notifyAll这些方法不在thread类里面？
一个很明显的原因是JAVA提供的锁是对象级的而不是线程级的，每个对象都有锁，通 过线程获得。如果线程需要等待某些锁那么调用对象中的wait()方法就有意义了。如果wait()方法定义在Thread类中，线程正在等待的是哪个锁 就不明显了。简单的说，由于wait，notify和notifyAll都是锁级别的操作，所以把他们定义在Object类中因为锁属于对象。
6.为什么wait和notify方法要在同步块中调用？
主要是因为Java API强制要求这样做，如果你不这么做，你的代码会抛出IllegalMonitorStateException异常。还有一个原因是为了避免wait和notify之间产生竞态条件。
7.Java中堆和栈有什么不同？
每个线程都有自己的栈内存，用于存储本地变量，方法参数和栈 调用，一个线程中存储的变量对其它线程是不可见的。而堆是所有线程共享的一片公用内存区域。对象都在堆里创建，为了提升效率线程会从堆中弄一个缓存到自己 的栈，如果多个线程使用该变量就可能引发问题，这时volatile 变量就可以发挥作用了，它要求线程从主存中读取变量的值。
8.什么是线程池？ 为什么要使用它？
创建线程要花费昂贵的资源和时间，如果任务来了才创建线程那么响应时间会变长，而且一个进程能创建的线程数有限。为了避免这些问题，在程序启动的时 候就创建若干线程来响应处理，它们被称为线程池，里面的线程叫工作线程。从JDK1.5开始，Java API提供了Executor框架让你可以创建不同的线程池。比如单线程池，每次处理一个任务；数目固定的线程池或者是缓存线程池（一个适合很多生存期短 的任务的程序的可扩展线程池）。
9.有三个线程T1，T2，T3，怎么确保它们按顺序执行？
在多线程中有多种方法让线程按特定顺序执行，你可以用线程类的join()方法在一个线程中启动另一个线程，另外一个线程完成该线程继续执行。为了确保三个线程的顺序你应该先启动最后一个(T3调用T2，T2调用T1)，这样T1就会先完成而T3最后完成。

##五、线程的优先级
Java线程的优先级是根据1-10的数来描述的.
Thread.min_Priority:最小
Thread.max_Priority:最大
Thread.norm_Priority=5
对线程优先级操作的方法:
void setPriority(int newpriority)设置线程的优先级
int getPriority:获取线程的优先级
注:线程的优先级是无法保证线程的执行顺序。只不过是获得时间片的概率更大一点。
